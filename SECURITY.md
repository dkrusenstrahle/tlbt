# Security Policy

## Supported versions

TLBT is currently pre-1.0. Security fixes are applied to the latest published version.

## Reporting a vulnerability

Please report suspected vulnerabilities privately.

- Do not open a public issue with exploit details.
- Include a clear description, impact assessment, and reproduction steps.
- If possible, include a proof-of-concept and suggested mitigation.

Maintainers will acknowledge reports promptly and work on a fix based on severity.

## Disclosure process

1. Report received and triaged.
2. Maintainers validate impact and prepare remediation.
3. A patched release is published.
4. Coordinated public disclosure is shared after a fix is available.
